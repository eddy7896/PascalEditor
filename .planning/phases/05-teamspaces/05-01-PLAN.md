    Update `apps/editor/prisma/schema.prisma` to close all three schema gaps from research:

    1. Add new enum `TeamRole` with values: OWNER, ADMIN, EDITOR, COMMENTER, VIEWER.

    2. Add `role TeamRole @default(VIEWER)` field to `TeamMember`. Keep existing `@@unique([teamId, userId])`.

    3. Add `avatarUrl String?` and `slug String?` to `Team` (slug optional, no unique constraint required for v1 — research recommendation).

    4. Add new model `TeamInviteToken`:
       ```prisma
       model TeamInviteToken {
         id              String    @id @default(cuid())
         token           String    @unique
         teamId          String
         inviteeEmail    String
         role            TeamRole
         createdByUserId String
         expiresAt       DateTime
         usedAt          DateTime?
         usedByUserId    String?
         team            Team      @relation(fields: [teamId], references: [id], onDelete: Cascade)
         createdAt       DateTime  @default(now())
         @@index([token])
         @@index([teamId])
         @@index([inviteeEmail])
       }
       ```

    5. Add `inviteTokens TeamInviteToken[]` relation field to `Team` model.

    6. Run `bunx prisma validate` to confirm schema is valid.
    7. Run `bunx prisma generate` to regenerate the client (output: `./generated-client`).

    DO NOT run `prisma migrate dev` — no local DB available (per STATE.md decisions, migrations deferred to deployment, validate + generate is the local pattern).
